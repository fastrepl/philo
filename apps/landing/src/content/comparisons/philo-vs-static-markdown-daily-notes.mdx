Philo stores daily notes as markdown files on disk, but edits them in memory as TipTap JSON.

## Quick take

Choose Philo when the center of gravity is the note itself. Choose static markdown daily notes when the center of gravity is somewhere else.

## Comparison table

| Decide this way if... | Philo | Static markdown daily notes |
| --- | --- | --- |
| You want planning context to persist across days | Built for it with timeline context and automatic carry-forward behavior. | Possible, but usually requires manual reconstruction and copying. |
| You want notes to stay as markdown on disk | Yes. Markdown remains the source of truth. | Yes. This is the baseline strength. |
| You want one-off tools inline in the note | Built for inline widgets and a reusable library. | Usually requires separate tools or custom setup. |

## What Philo is actually optimizing for

Keep daily planning in one timeline.

Philo is built to make daily notes lighter, not turn them into another subscription silo.
